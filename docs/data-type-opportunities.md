# Data-type opportunities — backlog

New data types that fit `pertype`'s compression model but aren't covered yet. Compiled from a
web-research sweep (Jun 2026). Ranked by fit × opportunity. Not yet validated — each entry needs
a measure-first benchmark against the named bar before building anything.

## The two win-modes (the screen)

- **Mode A — predict-per-type, then entropy-code.** A per-type predictor (spatial 2D/3D,
  temporal, multichannel cascade) leaves small residuals the context-adaptive arithmetic coder
  crushes. Wins on **smooth / structured / multichannel-correlated** numeric data.
- **Mode B — trained dictionary + LZ + arithmetic.** A model trained per file-type, amortized
  over **many small files of a known schema** (beats `zstd --train` on some types).
- **Columnar path.** De-interleave fixed-width record streams into per-field columns →
  per-column delta/Δ². Wins on tabular / record / telemetry streams.

**Meta-insight:** nearly every strong gap is the *same shape* — a smooth or multichannel-
correlated numeric field (or a schema-repetitive record stream) where the field bolted on a
**generic LZ / Blosc / gzip / LZ4** with no predictor. That is exactly the gap this codec closes.
The two biggest untapped veins: **multichannel int16/uint16 scientific time-series & imaging**
(Mode A) and **schema-repetitive small telemetry / record streams** (Mode B / columnar).

Recurring lessons: (1) **sparse / already-decorrelated data wants pure adaptive entropy coding,
not prediction** (cryo-EM imagecodec < ctxcoder; ephys cross-channel < per-channel); (2) **the
win needs clean low-entropy structure** — noisy (confocal) or float-mantissa-noisy (tomogram)
data is near-incompressible losslessly and no predictor helps; (3) **beating the field's
*storage* format (PNG/TIFF/EDF) is not beating the specialist** — pick the strongest real bar
(JPEG-LS/JXL, FLAC, zstd-`--train`) before claiming a win. Thermal-IR's spatial codec beat PNG
+27% but only *ties* JPEG-LS; its genuine win came from a lever the specialist still bar
**lacks** (an inter-frame temporal model), not from a better entropy coder — the same shape as
depth (beat PNG, but the honest margin is vs WebP-LL) and multispectral (the win is the 2D model
the per-band incumbent has no equivalent of); (4) **match the codec to each axis/array's
structure — route, don't unify.** A single file can hold one array that wants the columnar Δ²
lever and another that wants pure entropy: mass-spec's sorted m/z grid wins big with columnar
(+18% vs the HDF5 specialist) while its spiky intensity array *loses* to it (−15% vs zlib) and
must route to xz/shuffle — the same per-axis selectivity as multispectral (per-band 2D yes,
inter-band delta no) and hyperspectral (inter-band delta yes); (5) **redundancy that only exists
in a *transformed* domain is off-limits to a lossless codec.** MRI raw k-space genuinely has
multi-coil redundancy (the same anatomy seen by 18 coils) — but only after the FFT, in image
space; in the *raw* recorded float values the coils are decorrelated and the data is mantissa-
limited at a ~1.9× floor the generic byte-shuffle filter already hits. Reaching the redundancy
(coil-combine / low-rank / parallel imaging) is inherently lossy. "There is obvious redundancy"
is not a lossless lever unless it is exploitable in the stored representation without throwing
bits away — the same trap as the cryo-ET tomograms and LLM-weight mantissas; (6) **the columnar
Δ² lever needs a record *dominated* by sequential / monotonic / tick-grid integer fields — it
backfires on records dominated by high-cardinality *non-sequential* fields.** Financial ITCH won
+49% because sequential order IDs (Δ²→0) and monotonic ns timestamps *are* the record; NetFlow
flow records look superficially identical (fixed schema, timestamps, counters) but are dominated
by src/dst IPs, ports and byte-counters (~⅔ of the bytes) that have **no sequential structure**,
so per-column Δ/Δ² *inflates* them and byte-de-interleaving destroys the cross-record IP/label
repetition that plain `xz`/`zstd` already exploits — even an oracle per-column router lost to xz.
Screen a columnar candidate by asking *what fraction of the record bytes are monotonic/sequential*
before assuming the ITCH win transfers; (7) **a smooth numeric matrix stored row-major as TEXT is
a standing win — transpose to the correlated axis, predict, entropy-code.** Mocap BVH won +112%
over its as-shipped text+gzip and still +11.6% over a *sophisticated* generic transpose+Δ+xz
pipeline, because (a) the verbose text representation alone is a huge waste generic gzip can't
undo, (b) row-major order hides each channel's temporal continuity behind 95 unrelated channels,
and (c) once transposed and predicted, pertype's context-arithmetic coder beats xz on the
residual. This is the *positive* of lesson #6: where NetFlow's record was high-card non-sequential
(Δ backfires), a mocap matrix is **dominated by smooth temporally-correlated channels** (Δ/Δ²
collapses them) — and it reused the EEG/ECG per-channel predictor with zero new code, the hallmark
of every clean win here. The same shape recurs wherever a field ships smooth numeric arrays as text
or row-major-interleaved (telemetry CSV, simulation/animation curves, sensor logs): the lever is
*route to the correlated axis + reuse a predictor*, not a new codec.

Each is a measure-first task: grab the public data, compress with the existing codec (or a small
predictor tweak), and compare to the named bar before committing to build.
